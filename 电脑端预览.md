# StudyX 电脑端预览方案

## 方案一：浏览器直接预览（最简单）

### 适用场景
- 测试界面布局
- 测试CSS样式
- 测试JavaScript逻辑
- 无需测试Android原生功能（如TTS语音）

### 使用方法

1. **直接用浏览器打开HTML文件**
   ```
   打开文件：StudyX-Android/app/src/main/assets/www/index.html
   ```

2. **使用Chrome/Firefox的开发者工具**
   - 按 `F12` 打开开发者工具
   - 点击手机图标（或按 `Ctrl+Shift+M`）切换到设备模拟模式
   - 选择手机型号（如 iPhone 12 Pro、Pixel 5 等）
   - 可以模拟触摸操作、旋转屏幕等

### 优点
- ✅ 无需安装任何软件
- ✅ 修改后立即刷新就能看到效果
- ✅ 支持调试JavaScript

### 缺点
- ❌ 无法测试Android原生功能（语音播放、文件导出等）
- ❌ 部分交互可能与真机略有差异

---

## 方案二：本地HTTP服务器 + 手机同屏测试

### 适用场景
- 需要在真机上测试，但不想频繁打包APK
- 电脑和手机在同一WiFi下

### 使用方法

1. **启动本地服务器**
   ```bash
   cd StudyX-Android/app/src/main/assets/www
   python -m http.server 8080
   ```
   
   或者使用提供的脚本：
   ```bash
   双击运行：start_server.bat
   ```

2. **查看电脑IP地址**
   ```bash
   ipconfig
   ```
   找到类似 `192.168.1.100` 的地址

3. **手机浏览器访问**
   ```
   http://192.168.1.100:8080
   ```

4. **修改代码**
   - 在电脑上修改HTML/CSS/JS文件
   - 手机刷新页面即可看到效果

### 优点
- ✅ 真机测试，效果准确
- ✅ 无需打包APK
- ✅ 可以测试大部分功能

### 缺点
- ❌ 需要电脑和手机在同一网络
- ❌ 首次访问需要输入IP地址

## 方案三：Android Studio 模拟器

### 适用场景
- 需要完整测试Android原生功能
- 需要测试不同Android版本
- 正式发布前的最终测试

### 使用方法

1. **打开Android Studio**

2. **创建虚拟设备**
   ```
   Tools → Device Manager → Create Device
   ```

3. **选择设备类型**
   - Phone → Pixel 6（推荐）
   - 或选择其他型号

4. **选择系统镜像**
   - 下载 Android 11.0 (API 30) 或更高版本

5. **启动模拟器**
   - 点击 ▶️ 运行按钮

6. **安装APK**
   - 将APK文件拖到模拟器窗口
   - 或点击运行按钮直接部署

### 优点
- ✅ 最接近真机体验
- ✅ 支持所有Android原生功能
- ✅ 可以测试不同屏幕尺寸
- ✅ 支持调试日志

### 缺点
- ❌ 启动较慢（首次需要几分钟）
- ❌ 占用较多电脑资源
- ❌ 每次修改后需要重新安装APK（或使用热更新）

## 方案四：第三方Android模拟器（推荐）

### 推荐模拟器
1. **雷电模拟器** - 速度快，兼容性好
2. **夜神模拟器** - 功能丰富
3. **MuMu模拟器** - 网易出品，稳定
4. **蓝叠模拟器** - 老牌，性能好

### 使用方法

1. **下载安装**
   - 雷电：https://www.ldmnq.com/
   - 夜神：https://www.yeshen.com/

2. **安装APK**
   - 直接拖拽APK到模拟器窗口
   - 或点击APK文件选择用模拟器打开

3. **使用热更新**
   - 启动模拟器中的APP
   - 在电脑上修改代码并生成热更新包
   - APP自动检测并下载更新

### 优点
- ✅ 启动速度快
- ✅ 操作简单
- ✅ 支持键盘映射
- ✅ 可以多开

### 缺点
- ❌ 需要额外安装软件
- ❌ 部分模拟器可能有广告

## 快速选择建议

| 场景 | 推荐方案 |
|------|----------|
| 快速测试界面样式 | 方案一：浏览器预览 |
| 测试交互逻辑 | 方案一：浏览器 + 开发者工具 |
| 真机测试但不打包 | 方案二：HTTP服务器 |
| 测试完整功能 | 方案三/四：模拟器 |
| 正式发布前测试 | 方案三：Android Studio模拟器 |

## 小技巧

### 1. 浏览器自动刷新
安装 Live Server 插件（VS Code）：
- 修改代码后自动刷新浏览器
- 支持移动端模拟

### 2. 跨设备同步滚动
使用 Chrome DevTools 的 "Device Toolbar"：
- 可以同时查看不同尺寸的效果

### 3. 快速打包脚本
```bash
# 创建快速测试脚本 quick_test.bat
cd StudyX-Android
./gradlew.bat assembleDebug
copy app\build\outputs\apk\debug\app-debug.apk ..\test.apk
```
